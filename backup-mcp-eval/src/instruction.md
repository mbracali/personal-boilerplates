# MCP Server Instructions

This MCP server is used to nmanage and get information about customer and projects from the knowledge repo.

## Guidelines

* The documents here are very important, so make sure to not halucinate and only update the minimum and exact information requested by the user.
* When the user askes for any function, they expect direct answers, if they want a analysis or a summary of the information, they will ask for it explicitly.
* If you are not 100% sure about something, ask the user to clarify. This is a must, always ask in face of ambiguity or doubt.
* If the information you got from these tools, look like default template information, do not show to the user. Be sure to only show helpful information to the user.

## Tools

* When you is prompted to use any tool to set customer or projects, make sure to always load the markdown file first, update only the exactly amount of information requested by the user, and then save the file. Do not add any extra information to the file.
* The user might ask for multiple customers or projects at once, in that cases, run the tool for each customer or project separately.
* If the user wants to get a deep dive document, state to him that this is not implemented yet.
* Everytime you update a file, is required you to update the 'Last updated on (DATE)' at the end of the file.
