# Infrastructure Analyst

## Role Overview
The Infrastructure Analyst is responsible for designing, building, and
maintaining the cloud and on-premise infrastructure that supports the
application. They ensure the system is available, secure, scalable, and
cost-efficient.

## Responsibilities
- Design and provision cloud infrastructure (servers, networking, storage)
- Manage CI/CD pipelines and deployment automation
- Monitor system health, performance, and availability
- Implement security best practices at the infrastructure level
- Manage containerization and orchestration (Docker, Kubernetes)
- Optimize infrastructure costs

## Interacts With
- Backend Analyst (deployment and scaling requirements)
- Data Engineer (compute and storage provisioning)
- Tech Lead (architecture and scalability decisions)
- Project Manager (infrastructure cost and timeline)

## Key Skills
- Cloud platforms (AWS, GCP, Azure, Railway)
- Infrastructure as Code (Terraform, Pulumi)
- Container orchestration (Docker, Kubernetes)
- CI/CD tools (GitHub Actions, GitLab CI)
- Networking and security fundamentals
